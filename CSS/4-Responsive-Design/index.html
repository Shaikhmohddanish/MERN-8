Media Queries

Basic Media Query

Current Breakpoint:

/* Extra small devices (phones) */
.current-breakpoint::after {
    content: "Extra Small (< 576px)";
}

/* Small devices (tablets) */
@media (min-width: 576px) {
    .current-breakpoint::after {
        content: "Small (≥ 576px)";
    }
}

/* Medium devices (desktops) */
@media (min-width: 768px) {
    .current-breakpoint::after {
        content: "Medium (≥ 768px)";
    }
}

/* Large devices (desktops) */
@media (min-width: 992px) {
    .current-breakpoint::after {
        content: "Large (≥ 992px)";
    }
}

/* Extra large devices */
@media (min-width: 1200px) {
    .current-breakpoint::after {
        content: "Extra Large (≥ 1200px)";
    }
}

Resize your browser window to see the breakpoint indicator change. This demonstrates how media queries can detect the viewport size and apply different styles accordingly.

Responsive Units

Relative vs Absolute Units

Absolute Units (px)

This text is sized in pixels (fixed size).

200px × 100px

Relative Units (em)

This text is sized in em (relative to parent).

10em × 5em

Root Relative Units (rem)

This text is sized in rem (relative to root).

10rem × 5rem

.absolute-unit p {
    font-size: 16px;
}

.absolute-unit .unit-box {
    width: 200px;
    height: 100px;
}

.relative-unit p {
    font-size: 1em; /* 1em = parent's font size */
}

.relative-unit .unit-box {
    width: 10em; /* 10 × parent's font size */
    height: 5em; /* 5 × parent's font size */
}

.rem-unit p {
    font-size: 1rem; /* 1rem = root element's font size */
}

.rem-unit .unit-box {
    width: 10rem; /* 10 × root font size */
    height: 5rem; /* 5 × root font size */
}

Resize your browser window or change the font size in your browser settings to see how different units respond.

Viewport Units

50vw × 25vh
25vw × 50vh
50vmin × 50vmin

.vw-box {
    width: 50vw; /* 50% of viewport width */
    height: 25vh; /* 25% of viewport height */
}

.vh-box {
    width: 25vw; /* 25% of viewport width */
    height: 50vh; /* 50% of viewport height */
}

.vmin-box {
    width: 50vmin; /* 50% of the smaller viewport dimension */
    height: 50vmin; /* 50% of the smaller viewport dimension */
}

Viewport units size elements relative to the viewport dimensions. Resize your browser to see how these boxes adapt.

CSS Calc()

calc(50% - 20px)

.calc-box {
    width: calc(50% - 20px);
    height: calc(20vh + 100px);
    padding: calc(1rem + 5px);
}

The calc() function lets you perform calculations to determine CSS property values, combining different units.

Mobile-First Approach

Mobile-First vs Desktop-First

Mobile-First Approach

This element uses mobile-first media queries

Desktop-First Approach

This element uses desktop-first media queries


/* Mobile-First Approach */
.mobile-first .responsive-element {
    background-color: #f8d7da; /* Base style for mobile */
    padding: 10px;
    border-radius: 5px;
}

/* Enhance for larger screens */
@media (min-width: 768px) {
    .mobile-first .responsive-element {
        background-color: #d1ecf1;
        padding: 20px;
        display: flex;
        justify-content: center;
    }
}

/* Desktop-First Approach */
.desktop-first .responsive-element {
    background-color: #d1ecf1; /* Base style for desktop */
    padding: 20px;
    display: flex;
    justify-content: center;
    border-radius: 5px;
}

/* Simplify for smaller screens */
@media (max-width: 767px) {
    .desktop-first .responsive-element {
        background-color: #f8d7da;
        padding: 10px;
        display: block;
    }
}

Mobile-first starts with base styles for mobile devices and enhances for larger screens, while desktop-first does the opposite.

Progressive Enhancement Example

Basic Card

Content that all devices get


/* Base styles for all devices */
.feature-card {
    background: #f5f5f5;
    padding: 1rem;
    border-radius: 5px;
    border: 1px solid #ddd;
}

/* Medium screens get subtle enhancements */
@media (min-width: 768px) {
    .feature-card {
        padding: 1.5rem;
        box-shadow: 0 2px 5px rgba(0,0,0,0.1);
        transition: transform 0.3s ease;
    }
    
    .feature-card:hover {
        transform: translateY(-5px);
    }
}

/* Large screens get more enhancements */
@media (min-width: 992px) {
    .feature-card {
        display: grid;
        grid-template-columns: 2fr 1fr;
        grid-gap: 1rem;
        align-items: center;
    }
}

Progressive enhancement ensures all users get a functional experience, while users with more capable devices get enhanced features.

Responsive Typography

Fluid Typography with Viewport Units

Fluid Heading

This paragraph uses fluid typography that scales smoothly with the viewport size.


.fluid-heading {
    font-size: calc(1.5rem + 2vw);
    /* Base size (1.5rem) + viewport-based scaling (2vw) */
}

.fluid-text {
    font-size: calc(1rem + 0.5vw);
    /* Base size (1rem) + viewport-based scaling (0.5vw) */
}

Fluid typography uses calc() with a combination of fixed (rem) and viewport-relative (vw) units to create text that scales smoothly.

Media Query-Based Typography

Media Query Heading

This text uses different font sizes at different breakpoints.


/* Base size for mobile */
.mq-heading {
    font-size: 1.75rem;
}

.mq-text {
    font-size: 1rem;
    line-height: 1.5;
}

/* Medium devices */
@media (min-width: 768px) {
    .mq-heading {
        font-size: 2.25rem;
    }
    
    .mq-text {
        font-size: 1.125rem;
    }
}

/* Large devices */
@media (min-width: 992px) {
    .mq-heading {
        font-size: 2.75rem;
    }
    
    .mq-text {
        font-size: 1.25rem;
        line-height: 1.6;
    }
}

Using media queries for typography gives you precise control at each breakpoint, but creates "jumps" when resizing the browser.

Responsive Line Length

This paragraph demonstrates optimal line length for reading. Generally, 45-75 characters per line (including spaces) is considered ideal for readability. Too short and the eye has to travel back too frequently. Too long and it becomes difficult to track from the end of one line to the beginning of the next.


.optimal-reading {
    max-width: 65ch; /* 65 characters width */
    margin: 0 auto;
    line-height: 1.6;
}

@media (max-width: 768px) {
    .optimal-reading {
        max-width: 100%;
        padding: 0 1rem;
    }
}

The "ch" unit represents the width of the "0" character in the current font, making it perfect for controlling line length.

}

/* Mobile table styles */
@media (max-width: 768px) {
    .responsive-table thead {
        display: none; /* Hide table header */
    }
    
    .responsive-table, 
    .responsive-table tbody, 
    .responsive-table tr, 
    .responsive-table td {
        display: block;
        width: 100%;
    }
    
    .responsive-table tr {
        margin-bottom: 1rem;
        border: 1px solid #ddd;
        border-radius: 5px;
        padding: 0.5rem;
    }
    
    .responsive-table td {
        display: flex;
        justify-content: space-between;
        text-align: right;
        border-bottom: 1px solid #eee;
    }
    
    .responsive-table td::before {
        content: attr(data-label);
        font-weight: bold;
        text-align: left;
    }

Tables can be challenging on small screens. This technique transforms a traditional table into a card-like layout on mobile devices.

Paragraphs and Text Formatting in HTML

This page demonstrates how to work with paragraphs and apply various text formatting in HTML. Paragraphs are the basic building blocks of text content on the web.

Basic Paragraphs

The paragraph element (<p>) is the most fundamental text container in HTML. Browsers automatically add some margin space before and after paragraphs.

HTML Code Example

<p>This is the first paragraph. HTML paragraphs are defined with the p tag. Browsers automatically add some space (margin) before and after each paragraph.</p> <p>This is the second paragraph. Notice how there is space between this paragraph and the previous one. This happens automatically.</p>

Output

This is the first paragraph. HTML paragraphs are defined with the p tag. Browsers automatically add some space (margin) before and after each paragraph.

This is the second paragraph. Notice how there is space between this paragraph and the previous one. This happens automatically.

Note: Browsers automatically wrap text within paragraphs to fit the available space. They also collapse multiple spaces into a single space.

Line Breaks

When you need to create a line break without starting a new paragraph, use the <br> element. Unlike paragraphs, line breaks don't add extra spacing.

HTML Code Example

<p>This is the first line.<br> This is the second line.<br> This is the third line.</p>

Output

This is the first line.
This is the second line.
This is the third line.

Address Example with Line Breaks

<p> John Doe<br> 123 Main Street<br> Anytown, CA 12345<br> United States </p>

Output

John Doe
123 Main Street
Anytown, CA 12345
United States

Best Practice: Use <br> only when a line break is actually part of the content (like in addresses or poetry). Don't use it to create space between elements - use CSS margins for that purpose.

Horizontal Rules

The <hr> element creates a horizontal rule (line) that represents a thematic break between paragraph-level elements.

HTML Code Example

<p>This is the content above the horizontal rule.</p> <hr> <p>This is the content below the horizontal rule.</p>

Output

This is the content above the horizontal rule.

This is the content below the horizontal rule.

Combined Text Formatting

<p>The water molecule consists of two <strong><em>hydrogen</em></strong> atoms and one <strong><em>oxygen</em></strong> atom: H<sub>2</sub>O.</p>

The water molecule consists of two hydrogen atoms and one oxygen atom: H2O.

Semantic Meaning vs. Styling: Use these elements for their semantic meaning, not just for their visual effect. For example, use <strong> for important text, not just to make text bold. Use CSS for purely visual styling.

Whitespace Handling

HTML collapses multiple spaces, tabs, and line breaks into a single space. This behavior is called "whitespace collapsing".

HTML Code Example

<p>This text has many spaces but will display with single spaces.</p>

Output

This text has many spaces but will display with single spaces.

Preserving Whitespace with CSS

<p style="white-space: pre;">This text will preserve all spaces and line breaks.</p>

Output

This text will preserve all spaces and line breaks.

Alternative: For preserving whitespace, you can also use the <pre> element, which is designed specifically for preformatted text.

Non-Breaking Space

The non-breaking space (&nbsp;) is a special character that creates a space without allowing a line break at that position.

HTML Code Example

<p>Normal spaces allow line breaks between words.</p> <p>Non-breaking&nbsp;spaces&nbsp;keep&nbsp;words&nbsp;together.</p>

Output (resize window to see the effect)

Normal spaces allow line breaks between words.

Non-breaking spaces keep words together.

Common Uses for Non-Breaking Spaces

  • Preventing line breaks in phrases that should stay together (e.g., proper names, dates)
  • Creating indentation (though CSS is better for this)
  • Ensuring multiple spaces are displayed (since HTML collapses multiple regular spaces)
  • Preventing empty elements from collapsing

Best Practices for Text Content

  1. Use semantic elements that accurately describe the content's purpose
  2. Keep paragraphs focused on a single idea or topic
  3. Use formatting elements for their semantic meaning, not just for visual effects
  4. Keep line length manageable - ideally 50-75 characters per line for optimal readability
  5. Use CSS for styling rather than deprecated HTML attributes
  6. Consider readability - break up long text with headings, lists, and short paragraphs
  7. Ensure sufficient contrast between text and background colors
  8. Be consistent in formatting throughout your document
  9. Use proper punctuation and grammar to enhance readability
  10. Test content on various screen sizes to ensure it remains readable

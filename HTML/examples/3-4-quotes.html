HTML Quotes and Citations

HTML provides specialized elements for quoting content from other sources. These elements not only help format the content visually but also add semantic meaning to the quoted text.

Blockquotes (<blockquote>)

The <blockquote> element is used for extended quotations that form a separate paragraph or section.

Basic Blockquote

<blockquote> <p>The only way to learn a new programming language is by writing programs in it.</p> </blockquote>

The only way to learn a new programming language is by writing programs in it.

Blockquote with Citation Source

The cite attribute provides a URL to the source of the quotation:

<blockquote cite="https://www.goodreads.com/quotes/9168-the-only-way-to-learn-a-new-programming-language-is"> <p>The only way to learn a new programming language is by writing programs in it.</p> <footer>— Dennis Ritchie</footer> </blockquote>

The only way to learn a new programming language is by writing programs in it.

— Dennis Ritchie

Note: The cite attribute is not visible to users but provides semantic information for browsers, search engines, and assistive technologies. The actual citation source should be included in the content if you want it to be visible.

Styling Blockquotes with CSS

Blockquotes can be styled in various ways to make them stand out:

The only way to learn a new programming language is by writing programs in it.

— Dennis Ritchie

Nested Blockquotes

Blockquotes can be nested to show multiple levels of quotation:

<blockquote> <p>My favorite quote about programming is this:</p> <blockquote> <p>The only way to learn a new programming language is by writing programs in it.</p> <footer>— Dennis Ritchie</footer> </blockquote> <p>I've found this to be very true in my experience.</p> </blockquote>

My favorite quote about programming is this:

The only way to learn a new programming language is by writing programs in it.

— Dennis Ritchie

I've found this to be very true in my experience.

Inline Quotes (<q>)

The <q> element is used for shorter, inline quotations.

Basic Inline Quote

<p>As Steve Jobs said, <q>Design is not just what it looks like and feels like. Design is how it works.</q></p>

As Steve Jobs said, Design is not just what it looks like and feels like. Design is how it works.

Browsers typically render <q> elements with quotation marks. The style of quotation marks depends on the language set in the lang attribute of the HTML element:

French quotes

Inline Quote with Citation

<p>Steve Jobs once said, <q cite="https://www.brainyquote.com/quotes/steve_jobs_416927">Design is not just what it looks like and feels like. Design is how it works.</q></p>

Note: Like with blockquote, the cite attribute on <q> is not visible to users.

When to Use <q> vs. Manual Quotation Marks

While you could simply type quotation marks manually, using the <q> element provides these advantages:

  • Adds semantic meaning to the quote
  • Automatically uses the appropriate quotation mark style for the document's language
  • Allows for the inclusion of the citation source via the cite attribute
  • Makes it easier to style all quotes consistently via CSS

Citations (<cite>)

The <cite> element is used to represent the title of a creative work (e.g., a book, movie, article, poem, etc.).

Basic Citation Example

<p>My favorite book is <cite>The Lord of the Rings</cite> by J.R.R. Tolkien.</p>

My favorite book is The Lord of the Rings by J.R.R. Tolkien.

Browsers typically render <cite> elements in italics.

Citations in Blockquotes

The <cite> element can be used together with <blockquote> to properly attribute a quote:

<blockquote> <p>In a hole in the ground there lived a hobbit. Not a nasty, dirty, wet hole, filled with the ends of worms and an oozy smell, nor yet a dry, bare, sandy hole with nothing in it to sit down on or to eat: it was a hobbit-hole, and that means comfort.</p> <footer>— <cite>The Hobbit</cite> by J.R.R. Tolkien</footer> </blockquote>

In a hole in the ground there lived a hobbit. Not a nasty, dirty, wet hole, filled with the ends of worms and an oozy smell, nor yet a dry, bare, sandy hole with nothing in it to sit down on or to eat: it was a hobbit-hole, and that means comfort.

— The Hobbit by J.R.R. Tolkien

What Should Be Cited

According to the HTML specification, the <cite> element should be used for titles of:

  • Books
  • Movies and TV shows
  • Plays and musicals
  • Artworks
  • Articles and papers
  • Websites and web pages
  • Video games
  • Songs and albums
  • Other creative works

Common Misconception: The <cite> element is not meant for citing people's names (although some earlier HTML specifications suggested this use). It should be used for the titles of works, not the creators of those works.

Related HTML Elements

The <address> Element

While not strictly a quotation element, <address> is often used in conjunction with citations to provide contact information for the author of a section or article:

The <dfn> Element

The <dfn> element is used to mark the defining instance of a term:

<p><dfn>HTML</dfn> (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser.</p>

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser.

Accessibility and SEO Considerations

Accessibility Benefits

  • Semantic Meaning: Screen readers can identify quotes and citations, providing context to users
  • Proper Attribution: Clearly identifies the source of quoted material
  • Navigational Aids: Some assistive technologies allow users to navigate between different types of content, including quotes

SEO Benefits

  • Content Structure: Proper use of quotation elements helps search engines understand the structure of your content
  • Citation Context: The cite attribute can help establish relationships between your content and the cited sources
  • Rich Snippets: In some cases, search engines may use blockquote content in search results

Best Practices

  1. Use <blockquote> for extended quotes that form a separate paragraph or section
  2. Use <q> for short, inline quotes rather than typing quotation marks manually
  3. Use <cite> for titles of works, not for people's names
  4. Include the cite attribute when the source URL is available
  5. Make the citation visible in the content (usually in a footer) if users need to see it
  6. Nest quotes properly when quoting someone who is quoting someone else
  7. Style quotes to make them stand out from the surrounding text
  8. Don't use quotation elements just for their visual effects (like indentation)
  9. Consider internationalization - the <q> element adapts to the document language
  10. Combine with other semantic elements when appropriate (<footer>, <address>, etc.)

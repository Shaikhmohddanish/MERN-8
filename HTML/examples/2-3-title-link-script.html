Title, Link, and Script Elements

This example demonstrates the three important head elements: title, link, and script.

The <title> Element

The <title> element defines the title of the HTML document, which appears in:

  • Browser tabs
  • Bookmarks/favorites
  • Search engine results
  • Browser history
<title>Title, Link, and Script Elements | HTML Guide</title>

Best Practices for Titles:

  • Keep them concise but descriptive (50-60 characters)
  • Include important keywords near the beginning
  • Make each page's title unique
  • Consider using a pattern (e.g., "Page Name | Site Name")

The <link> Element

The <link> element defines the relationship between the current document and an external resource. It's most commonly used to link to stylesheets.

Common Uses of the Link Element

Purpose Code Example
External CSS <link rel="stylesheet" href="styles.css">
Favicon <link rel="icon" href="favicon.ico" type="image/x-icon">
Web Fonts <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Roboto">
Preconnect <link rel="preconnect" href="https://example.com">
Canonical URL <link rel="canonical" href="https://example.com/page">
Alternate Format <link rel="alternate" type="application/rss+xml" href="rss.xml">
Preload Resources <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

Key Attributes of the Link Element

  • rel: Specifies the relationship between the current document and the linked resource
  • href: Specifies the location (URL) of the linked resource
  • type: Specifies the MIME type of the linked resource
  • media: Specifies which media/device the linked resource is optimized for
  • sizes: Specifies the size of icons for visual media
  • crossorigin: Specifies how the element handles crossorigin requests

Performance Tip: Use rel="preload" for critical resources like fonts and hero images to prioritize their loading.

The <script> Element

The <script> element is used to embed or reference executable code, typically JavaScript.

Ways to Include JavaScript

1. External JavaScript File

<script src="script.js"></script>

2. Inline JavaScript

<script> function greet() { alert("Hello, world!"); } </script>

Script Loading Strategies

Strategy Code Example Description
Regular <script src="script.js"></script> Blocks HTML parsing until the script is loaded and executed
Defer <script src="script.js" defer></script> Downloads during HTML parsing, executes after parsing completes
Async <script src="script.js" async></script> Downloads asynchronously, executes as soon as available
Module <script type="module" src="module.js"></script> Treated as a JavaScript module with its own scope

Key Attributes of the Script Element

  • src: Specifies the URL of an external script file
  • type: Specifies the MIME type of the script
  • defer: Script executes after the document has been parsed
  • async: Script executes asynchronously as soon as it's available
  • integrity: Contains inline metadata used to verify the resource
  • crossorigin: Specifies how the element handles crossorigin requests
  • nomodule: Script should not be executed in browsers that support ES modules

Best Practice: Place scripts at the end of the <body> element or use defer attribute for better page loading performance.

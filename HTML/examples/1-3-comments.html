HTML Comments: The Developer's Invisible Notes

Understanding how to use comments in HTML documents

What Are HTML Comments?

HTML comments are notes that are invisible to users viewing your page in a browser. They help developers by:

  • Explaining sections of code
  • Making notes for future reference
  • Temporarily disabling code without deleting it
  • Organizing large documents
  • Communicating with team members

Comments are written using the following syntax:

<!-- This is an HTML comment -->

Everything between <!-- and --> is treated as a comment and not displayed on the page.

Comment Examples

1. Single-line comments

<!-- This is a single-line comment -->

2. Multi-line comments

<!-- This is a multi-line comment. It can span across several lines. Use this for longer explanations. -->

3. Commenting out code

<!-- <div class="feature"> <h2>Coming Soon</h2> <p>This feature is still in development.</p> </div> -->

Result: The commented-out code won't be displayed or processed by the browser.

4. Section dividers

<!-- ========== HEADER SECTION ========== --> <header>...</header> <!-- ========== MAIN CONTENT ========== --> <main>...</main>

5. Todo notes

<!-- TODO: Add responsive navigation menu --> <nav>...</nav>

Comment Best Practices

Do's:

  • Use comments to explain complex code sections
  • Comment code that might not be obvious to other developers
  • Use comments to organize large documents into logical sections
  • Keep comments up-to-date when changing code
  • Use consistent commenting style throughout a project

Don'ts:

  • Don't overuse comments for obvious code
  • Don't leave outdated comments that no longer match the code
  • Don't put sensitive information in comments (they're visible in source code)
  • Don't nest comments (comments inside comments don't work in HTML)
  • Don't rely on comments instead of writing clear, self-explanatory code
<!-- This doesn't work: <!-- Nested comment --> -->

Comments vs. Hidden Content

It's important to understand the difference between HTML comments and other ways to hide content:

Method Visible in Browser Visible in Source Code Use Case
<!-- Comments --> No Yes Developer notes, documentation
display: none; (CSS) No Yes Dynamic content that may be shown later
visibility: hidden; (CSS) No (but takes up space) Yes Elements that should reserve their space
hidden attribute No Yes Elements that should be hidden by default

Note: Remember that HTML comments are not secure for hiding sensitive information, as they're still visible to anyone who views your page source.

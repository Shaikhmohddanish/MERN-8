Preformatted Text and Code in HTML

HTML provides specialized elements for displaying preformatted text and code. These elements are essential for technical documentation, tutorials, and any content that requires precise formatting or displays programming code.

Preformatted Text: The <pre> Element

The <pre> element preserves both spaces and line breaks exactly as they appear in the HTML source code. This makes it perfect for displaying content where whitespace is significant.

Basic Preformatted Text

<pre> This text maintains its exact formatting, including spaces and line breaks. </pre> 
    This text maintains
        its exact formatting,
            including spaces
        and line breaks.

Notice how the indentation and line breaks in the HTML source are preserved in the browser output. This is different from normal HTML, where multiple whitespace characters are collapsed into a single space.

ASCII Art with Preformatted Text

The <pre> element is perfect for displaying ASCII art:

    .--.
   |o_o |
   |:_/ |
  //   \ \
 (|     | )
/'\_   _/`\

Note: While <pre> preserves whitespace, it still interprets HTML entities and tags. If you need to display HTML tags as text, you'll need to use character entity references (e.g., &lt; for < and &gt; for >) or combine it with the <code> element.

Overflow Handling

When preformatted text is too wide for its container, it can create horizontal scrollbars. You can control this with CSS:

pre { overflow-x: auto; /* Adds horizontal scrollbar when needed */ white-space: pre-wrap; /* Allows text to wrap to next line */ word-wrap: break-word; /* Breaks long words to prevent overflow */ }

With overflow-x: auto:

This is a very long line of text that will extend beyond the width of its container and create a horizontal scrollbar rather than wrapping to the next line. This demonstrates how overflow-x: auto works with preformatted text.

With white-space: pre-wrap:

This is a very long line of text that will wrap to the next line when it reaches the edge of its container. This demonstrates how white-space: pre-wrap works with preformatted text.

Code Elements: <code>

The <code> element is used to represent computer code. By default, browsers display <code> elements in a monospace font.

Inline Code

<p>The <code>console.log()</code> function outputs a message to the console.</p>

The console.log() function outputs a message to the console.

Code Blocks with <pre> and <code>

For multi-line code blocks, combine <pre> and <code>:

<pre><code> function greet(name) { console.log("Hello, " + name + "!"); } greet("World"); </code></pre> 

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet("World");

Best Practice: The combination of <pre> and <code> is recommended for code blocks. The <pre> preserves formatting, while the <code> adds semantic meaning.

Displaying HTML Code

When displaying HTML code, you need to replace angle brackets with their entity references to prevent the browser from interpreting them as actual HTML tags:

<pre><code> &lt;div class="container"&gt; &lt;h1&gt;Hello, World!&lt;/h1&gt; &lt;p&gt;This is a paragraph.&lt;/p&gt; &lt;/div&gt; </code></pre> 

<div class="container">
    <h1>Hello, World!</h1>
    <p>This is a paragraph.</p>
</div>

Syntax Highlighting

While HTML doesn't provide built-in syntax highlighting, you can create highlighted code blocks using CSS classes:


function calculateArea(radius) {
    // Calculate the area of a circle
    const pi = 3.14159;
    return pi * radius * radius;
}

let result = calculateArea(5);
console.log("The area is: " + result);

For production websites, you might want to use a dedicated syntax highlighting library like Prism.js or Highlight.js.

Line Numbers

Adding line numbers to code blocks can make it easier to reference specific lines:

function calculateArea(radius) {
    // Calculate the area of a circle

let result = calculateArea(5);

Other Code-Related Elements

HTML includes several specialized elements for different types of computer code and inputs:

The <kbd> Element

Represents keyboard input:

The <samp> Element

Represents sample output from a computer program:

<p>The program returned: <samp>Error: File not found</samp></p>

The program returned: Error: File not found

Math Formula Example

Quadratic Formula

The quadratic formula is used to solve equations in the form ax2 + bx + c = 0, where a ≠ 0.

The formula is:

x = (-b ± √(b2 - 4ac)) / (2a)

To implement this in JavaScript:


function solveQuadratic(a, b, c) {
    const discriminant = b * b - 4 * a * c;
    
    if (discriminant < 0) {
        return "No real solutions";
    } else if (discriminant === 0) {
        const x = -b / (2 * a);
        return `One solution: x = ${x}`;
    } else {
        const x1 = (-b + Math.sqrt(discriminant)) / (2 * a);
        const x2 = (-b - Math.sqrt(discriminant)) / (2 * a);
        return `Two solutions: x = ${x1} or x = ${x2}`;
    }
}

Accessibility Considerations

When using preformatted text and code elements, keep these accessibility considerations in mind:

Screen Readers

  • Screen readers typically announce when they encounter code elements, providing context to users
  • Preformatted text may be read character by character, which can be confusing for long blocks
  • Adding appropriate context before code blocks helps users understand what follows

Color Contrast

  • If you use syntax highlighting, ensure there's sufficient contrast between the text and background
  • The WCAG recommends a contrast ratio of at least 4.5:1 for normal text

Scrollable Content

  • When making code blocks scrollable, ensure they can be navigated with a keyboard
  • Consider using tabindex="0" to make scrollable containers focusable

Alternative Formats

  • For complex ASCII art, consider providing an alternative description
  • For important code examples, consider explaining the code in plain text as well

Best Practices

  1. Combine <pre> and <code> for code blocks to get both formatting and semantic benefits
  2. Escape HTML characters when displaying HTML code (&lt; for <, &gt; for >)
  3. Add line breaks in your source code to improve readability
  4. Consider syntax highlighting for better readability of code examples
  5. Use responsive design techniques to handle overflow on small screens
  6. Use the appropriate semantic element for the type of content (<kbd>, <samp>, <var>)
  7. Provide context before code blocks to explain what they demonstrate
  8. Keep code examples concise and focused on the concept being explained
  9. Consider using line numbers for longer code blocks or when you need to reference specific lines
  10. Test with screen readers to ensure accessibility

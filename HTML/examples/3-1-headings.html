HTML Headings: h1 to h6 Elements

This page demonstrates the different heading levels in HTML and explains when and how to use them effectively.

Understanding HTML Heading Elements

HTML provides six levels of headings, from <h1> to <h6>, which are used to structure content in a hierarchical manner.

<h1>Main Title of the Page</h1>
<h2>Major Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Lowest-level Heading</h6>

Visual Hierarchy of Headings

By default, browsers render headings with the following characteristics:

Heading Default Size Default Style Importance
<h1> 2em (32px) Bold Highest
<h2> 1.5em (24px) Bold Very High
<h3> 1.17em (18.72px) Bold High
<h4> 1em (16px) Bold Medium
<h5> 0.83em (13.28px) Bold Low
<h6> 0.67em (10.72px) Bold Lowest

Note: While headings have default visual styles, their primary purpose is to create a semantic structure. Always use CSS to modify the appearance of headings, not to determine which heading level to use.

The Document Outline

Headings create a document outline or table of contents for your page. This outline is used by:

  • Screen readers to help visually impaired users navigate the page
  • Search engines to understand the structure and importance of content
  • Browser features like "Reader Mode" to identify important sections
  • Browser extensions that generate tables of contents

Proper Heading Hierarchy

Always maintain a proper hierarchy in your headings. Don't skip levels (like going from <h2> directly to <h4>) as this breaks the logical structure of your document.

Common Heading Mistakes
  • Using headings for styling purposes rather than structure
  • Skipping heading levels (e.g., h1 → h3 → h5)
  • Using multiple h1 elements on a single page
  • Not using headings at all for major sections
Additional Resources

For more information on proper heading usage, consult the MDN Web Docs.

Headings and SEO

Headings play a crucial role in search engine optimization (SEO) as they help search engines understand the content and structure of your page.

SEO Best Practices for Headings

  1. Use keywords strategically in your headings, especially in h1 and h2 elements
  2. Keep headings descriptive but concise - aim for under 60 characters
  3. Ensure heading text accurately reflects the content that follows
  4. Use only one h1 element per page, typically containing your primary keyword
  5. Structure headings hierarchically to create a logical outline

SEO Tip: While it's important to include keywords in headings, avoid "keyword stuffing" which can negatively impact your search ranking. Always prioritize readability and user experience.

Styling Headings with CSS

While headings have default styles in browsers, you can customize their appearance using CSS.

Basic Heading Styles

h1 {
  color: #2c3e50;
  font-size: 2.5em;
  font-weight: 700;
  text-align: center;
  margin-bottom: 30px;
}

h2 {
  color: #3498db;
  font-size: 1.8em;
  border-bottom: 2px solid #3498db;
  padding-bottom: 10px;
}

Advanced Styling Techniques

You can create more distinctive headings using CSS features like:

  • Gradients
  • Text shadows
  • Letter spacing
  • Custom fonts
  • Decorative underlines
  • Pseudo-elements for additional visual effects

Accessibility Reminder: When styling headings, ensure there is sufficient contrast between the text and background colors. The WCAG recommends a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text.

Accessibility Considerations

Properly structured headings are essential for accessibility, particularly for users of screen readers and other assistive technologies.

How Screen Readers Use Headings

Screen reader users often navigate pages by jumping from heading to heading. Many screen readers provide shortcuts to:

  • Jump to the next/previous heading
  • Jump to headings of a specific level
  • Generate a list of all headings on the page

This makes proper heading structure crucial for providing an accessible experience.

ARIA Roles for Headings

In some cases, you might need to use ARIA roles to clarify heading levels:

<div role="heading" aria-level="2">This acts as an h2 heading</div>

However, it's always better to use native heading elements (<h1> through <h6>) when possible.
